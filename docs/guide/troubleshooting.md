# Troubleshooting & FAQ

This guide covers common errors and questions you might encounter when using EnvKnit.

---

## 1. CLI and Executable Issues

### `command not found: pytest` (or `black`, `mypy`, `ruff`)
**Symptom:** Running `envknit run -- pytest` fails with a command not found error.

**Cause:** EnvKnit does not generate `bin/` or `Scripts/` wrappers for Python tools like a traditional virtual environment does. This is intentional to allow multiple versions to coexist globally.

**Solution:** Invoke the tool as a Python module using the `-m` flag.
```bash
# Wrong
envknit run -- pytest

# Correct
envknit run -- python -m pytest
```
Read the full [CLI Scripts Guide](cli-scripts.md) for more details.

## 2. Python API and Import Issues

### `CExtensionError` when using `envknit.use()`
**Symptom:** You get an `envknit.isolation.import_hook.CExtensionError: ImportError` when trying to use `with envknit.use("numpy", ...):`.

**Cause:** The `envknit.use()` context manager dynamically routes imports within the same Python process. However, CPython does not support loading multiple versions of the same C extension (like `numpy` or `pandas`) in the same process.

**Solution:** Use the `envknit.worker()` API instead, which runs the C-extension package in an isolated subprocess.
```python
# Change this:
with envknit.use("numpy", "1.24.0"):
    import numpy as np

# To this:
with envknit.worker("numpy", "1.24.0") as np:
    arr = np.zeros(100)
```
Read more in the [Python API Guide](python-api.md).

### `ModuleNotFoundError: No module named 'X'`
**Symptom:** Your Python script raises a `ModuleNotFoundError` even though the package is listed in `envknit.yaml`.

**Cause:** 
1. The packages have not been installed to the global store.
2. Or, you are running standard `python app.py` instead of `envknit run -- python app.py`.

**Solution:**
Make sure you have run the installation:
```bash
envknit lock
envknit install
```
Then, ensure you are running your code through the CLI so that the `PYTHONPATH` is injected:
```bash
envknit run -- python app.py
```

## 3. Schema and Resolution Issues

### `SchemaVersionError`
**Symptom:** `envknit.isolation.import_hook.SchemaVersionError: ValueError`.

**Cause:** Your `envknit.lock.yaml` was generated by a newer version of the Rust CLI than your installed Python library supports.

**Solution:** Upgrade your Python `envknit` package.
```bash
pip install --upgrade envknit
```

### Dependency Resolution Fails (`envknit lock`)
**Symptom:** The lock command fails with a SAT solver or version conflict error.

**Cause:** You have requested package versions that cannot coexist (e.g., `requests>=2.31` and another package that requires `requests<2.30`).

**Solution:**
1. Check the error output to see which packages are conflicting.
2. Relax the version constraints in `envknit.yaml`.
3. If the packages are completely incompatible, consider placing them in separate [Environments](environments.md).
